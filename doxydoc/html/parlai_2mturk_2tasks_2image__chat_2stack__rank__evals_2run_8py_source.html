23  Human Evaluation of various responses to comments on images.
24 
25  A turker is shown an image and some dialog history. Then, the
26  turker is asked to choose which response they think is more engaging.
27 
28  If no `--eval-data-path` is given, the data from the original
29  Image-Chat dataset is used.
30 
31  To use your own data, please specify `--eval-data-path`, a path to an
32  appropriate json file with a list of examples, where each example
33  has the following structure:
34  {
35  'image_hash': <hash of image>,
36  'dialog': [(personality, text), ...] - list of personality, text tuples
37  'personality': <personality of responses to compare>
38  '<compare_key_1>': <first response to compare>,
39  '<compare_key_2>': <second option to compare>,
40  .
41  .
42  .
43  }
44  Note that compare_key_1 and compare_key_2 can be any field, as long as they
45  map to a string response.
46 
47  Example Scenario:
48  Suppose you have the original Image-Chat dataset, and
49  you would like to compare the outputs of your model called `model`.
50 
51  Your data may look like the following:
52  [{
53  'image_hash': hashforimageofcat,
54  'dialog': [
55  ('Sweet', 'What a cute cat!'),
56  ('Neutral', 'Just looks like a plain cat to me')

58  'personality': 'Sweet',

60  'model_comment': 'You'll love it if you pet it!' # Model Comment
61  }, ...]
62 
63  Thus, you would specify `-ck1 comment -ck2 model_comment` to evaluate
64  the outputs of the model vs. the human comments from Personality-Captions
