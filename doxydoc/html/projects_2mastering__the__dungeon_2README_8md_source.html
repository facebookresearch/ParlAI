33 ### Data Organization
34 
35 The dataset is organized as follows. `data/graph_world2` contains the pilot study data, where each file ending with `.pkl` is a pickle file of an example. `data/graph_world2_v*_r*` contains a pickle file `filtered_data_list.pkl` storing the data collected in a specific round of a specific setting. The number after `r` indicates the round index. The number after `v` indicates the setting, where `v13` means `MTD limit`, `v14` means `MTD`, `v15` means `MTD limit w/o model`, and `BASELINE_2` means the baseline. For example, `data/graph_world2_v15_r2/filtered_data_list.pkl` contains the data collected in the second round of the setting `MTD limit w/o model`.

47 ## Training and Evaluation
48 
49 There are three steps: 1) start a GPU placeholder; 2) run the training and evaluation jobs; 3) terminate the GPU placeholder. We will illustrate the usage first on one single GPU, and later on a cluster with Slurm installed.
50 
51 ### Start GPU Placeholder
52 
53 Open a new window (e.g. a screen session), and go to the ParlAI root directory.
54 ```
55 cd projects/mastering_the_dungeon/projects/graph_world2
56 python train.py --job_num 0
57 ```
58 
59 This will start a GPU placeholder that accepts new training jobs on the fly.

61 ### Run Training Jobs
62 
63 Open another window (e.g. another screen session), and go to the ParlAI root directory.

65 #### Training
66 ```
67 cd projects/mastering_the_dungeon/mturk/tasks/MTD
68 python run.py --train [--seq2seq]
69 ```
70 
71 This will train the models using AC-Seq2Seq (by default) or Seq2Seq (when seq2seq is specified).

109 ### Running on GPU Cluster
110 
111 There are a few changes to be made when running on a GPU cluster. In this section we assume [Slurm](https://slurm.schedmd.com/) is installed on the cluster.
112 
113 When creating the GPU placeholder, we use the following commands:
114 ```
115 cd projects/mastering_the_dungeon/projects/graph_world2
116 python gen_sbatch_script.py --num_gpus <num_gpus> --slurm
117 ./batch_holder.sh
118 ```
119 where `num_gpus` is the number of GPUs to use.
120 
121 For training and evaluation, we need to add an additional option `--num_machines` to every command. For example, training now becomes:
122 ```
123 cd projects/mastering_the_dungeon/mturk/tasks/MTD
124 python run.py --train --num_machines <num_gpus> [--seq2seq]
125 ```
126 
127 To terminate the GPU placeholder, simply cancel all jobs (this assumes you are not running other jobs using Slurm)
128 ```
129 scancel -u <my_username_on_slurm>
130 ```
