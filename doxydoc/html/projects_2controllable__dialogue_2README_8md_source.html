11 ## Abstract
12 
13 A good conversation requires balance -- between simplicity and detail; staying
14 on topic and changing it; asking questions and answering them. Although
15 dialogue agents are commonly evaluated via human judgments of overall quality,
16 the relationship between quality and these individual factors is less
17 well-studied. In this work, we examine two controllable neural text generation
18 methods, conditional training and weighted decoding, in order to control four
19 important attributes for chitchat dialogue: repetition, specificity,
20 response-relatedness and question-asking. We conduct a large-scale human
21 evaluation to measure the effect of these control parameters on multi-turn
22 interactive conversations on the PersonaChat task. We provide a detailed
23 analysis of their relationship to high-level aspects of conversation, and show
24 that by controlling combinations of these variables our models obtain clear
25 improvements in human quality judgments.

40 # Code Instructions
41 Once you have [installed ParlAI](https://github.com/facebookresearch/ParlAI/#installing-parlai),
42 follow the instructions below.

44 ## Download the data
45 
46 Running the commands to train or chat with the models will automatically download
47 the data for you. Alternatively, you can manually download the data by
48 running `python projects/controllable_dialogue/tasks/build.py`. This will download
49 the following files to `data/controllable_dialogue`:
50 
51 - `train.txt`: This is Convai2 training data, with extra annotations for three
52  CT controllable attributes (`question`, `lastuttsim`, `avg_nidf`). It is in
53  parlai format.
54 - `valid.txt`: Similarly to train.txt.
55 - `arora.pkl`: This is a pickle file containing information necessary to
56  compute Arora-style sentence embeddings, needed for the response-relatedness
57  control methods.
58 - `word2count.pkl`: This is a pickle file containing information necessary to
59  compute NIDF measures, needed for the specificity control methods.
60 - `personas_validation.txt`: This file contains all the ConvAI2 validation set
61 personas, provided for convenience (useful for talking to the model interactively).
62 - `ConvAI2_parlaiformat/`:
63  - `train.txt`: This is the ConvAI2 training set
64  (`data/ConvAI2/train_self_original_no_cands.txt`) converted to parlai
65  format.
66  - `valid.txt`: This is the ConvAI2 validation set
67  (`data/ConvAI2/valid_self_original_no_cands.txt`) converted to parlai
68  format.
69 - `wordstat_files/`: This directory contains json files with generated output and
70 automatic metrics computed for the various pretrained models.
71 - `evaluation_logs/`: This directory contains logs and evaluations from the human
72  evaluations.

74 ### (Alternatively) Making the data yourself
75 
76 For reproducibility, in this section we provide the commands to create
77 the data yourself.
78 
79 _Note: Due to changes in ParlAI, there might be some small differences between
80 the generated files obtained in this section, and the downloaded files in the
81 previous section._
82 
83 First, convert the ConvAI2 data to ParlAI format:
84 
85  mkdir -p data/controllable_dialogue/ConvAI2_parlaiformat
86 
87  python parlai/scripts/convert_data_to_parlai_format.py \
88  --task convai2:SelfOriginal:no_cands \
89  --datatype train:ordered \
90  --outfile data/controllable_dialogue/ConvAI2_parlaiformat/train.txt
91 
92  python parlai/scripts/convert_data_to_parlai_format.py \
93  --task convai2:SelfOriginal:no_cands \
94  --datatype valid \
95  --outfile data/controllable_dialogue/ConvAI2_parlaiformat/valid.txt
96 
97 Next, create `word2count.pkl`:
98 
99  python projects/controllable_dialogue/controllable_seq2seq/nidf.py
100 
101 This will create a file called `word2count.pkl` in your `data/controllable_dialogue`
102 directory. It might take a while, especially the part when it goes through the
103 Twitter dataset counting words.
104 
105 Next, create `arora.pkl`:
106 
107  python projects/controllable_dialogue/controllable_seq2seq/arora.py
108 
109 This will create a file called `arora.pkl` in your `data/controllable_dialogue`
110 directory. It might take a while - in particular, if necessary it will download
111 GloVe vectors and store them in `ParlAI/data/models/glove_vectors`.
112 
113 Next, create `data/controllable_dialogue/train.txt` and `valid.txt`:
114 
115  python projects/controllable_dialogue/make_control_dataset.py \
116  --fromfile_datapath data/controllable_dialogue/ConvAI2_parlaiformat/train.txt \
117  --outfile data/controllable_dialogue/train.txt \
118  --controls question,lastuttsim,avg_nidf
119 
120  python projects/controllable_dialogue/make_control_dataset.py \
121  --fromfile_datapath data/controllable_dialogue/ConvAI2_parlaiformat/valid.txt \
122  --outfile data/controllable_dialogue/valid.txt \
123  --controls question,lastuttsim,avg_nidf
124 
125 This will create files called `train.txt` and `valid.txt` in your
126 `data/controllable_dialogue` directory.

128 ## The pretrained models
129 
130 Running the commands in the next section to chat with the pretrained models will
131 automatically download them for you. In `data/models/controllable_dialogue` you will
132 find the following models, along with their `.opt` files:
133 
134 - `twitter_pretrained_baseline`: A seq2seq model trained on the Twitter dataset.
135 - `convai2_finetuned_baseline`: The `twitter_pretrained_baseline` model, after
136  fine-tuning on the ConvAI2 dataset.
137 - `control_avgnidf10b10e`: The `convai2_finetuned_baseline` model, after adding
138  parameters for CT specificity control (10 buckets, embedding size 10), and
139  fine-tuned on the ConvAI2 dataset with loss\_CT as described in Section 5.1 of
140  the paper.
141 - `control_questionb11e10`: Similarly to `control_avgnidf10b10e`, except this
142  is CT question-asking control (11 buckets, embedding size 10).
143 
144 The directory also contains a dictionary file:
145 
146 - `dict_twit30k_train_split`: This is the dictionary used for all models.

148 ## Chat with the pretrained models
149 
150 This section provides the commands to talk to the model configurations
151 described in the paper. You can refer to Table 5 in the paper to see how these
152 commands correspond to the configurations described there.
153 
154 Running any of these commands will also download the pretrained models, if necessary.
155 
156 **Talk to the greedy search baseline model:**
157 
158  python projects/controllable_dialogue/interactive.py \
159  -mf models:controllable_dialogue/convai2_finetuned_baseline \
160  --beam-size 1
161 
162 **Talk to the beam search baseline model:**
163 
164  python projects/controllable_dialogue/interactive.py \
165  -mf models:controllable_dialogue/convai2_finetuned_baseline
166 
167 This setting uses beam size 20 by default.
168 
169 **Talk to the repetition-controlled (WD) baseline:**
170 
171  python projects/controllable_dialogue/interactive.py \
172  -mf models:controllable_dialogue/convai2_finetuned_baseline \
173  -wd extrep_2gram:-3.5,extrep_nonstopword:-1e20,intrep_nonstopword:-1e20
174 
175 You can change the weights for these three WD repetition features to be any
176 real number (positive or negative). Here `-1e20` represents -infinity. In
177 addition, there are other repetition WD features you can use if you wish: see
178 the keys of `WDFEATURE2UPDATEFN` in `controllable_seq2seq/controls.py`.
179 
180 **Talk to the question-controlled CT model (with WD repetition control):**
181 
182  python projects/controllable_dialogue/interactive.py \
183  -mf models:controllable_dialogue/control_questionb11e10 \
184  -wd extrep_2gram:-3.5,extrep_nonstopword:-1e20,intrep_nonstopword:-1e20 \
185  --set-controls question:7
186 
187 Here `question:7` means the '70% questions' bucket.
188 You can set this anywhere between 0 and 10.
189 
190 To talk to the "z=10 (boost)" version mentioned in the paper:
191 
192  python projects/controllable_dialogue/interactive.py \
193  -mf models:controllable_dialogue/control_questionb11e10 \
194  -wd extrep_nonstopword:-1e20,intrep_nonstopword:-1e20 \
195  --set-controls question:10 --beam-reorder best_extrep2gram_qn
196 
197 **Talk to the specificity-controlled CT model (with WD repetition control):**
198 
199  python projects/controllable_dialogue/interactive.py \
200  -mf models:controllable_dialogue/control_avgnidf10b10e \
201  -wd extrep_2gram:-3.5,extrep_nonstopword:-1e20,intrep_nonstopword:-1e20 \
202  --set-controls avg_nidf:7
203 
204 Here `avg_nidf:7` means the 7th specificity bucket (where higher is more specific).
205 You can set this anywhere between 0 and 9.
206 
207 **Talk to the specificity-controlled WD model (with WD repetition control):**
208 
209  python projects/controllable_dialogue/interactive.py \
210  -mf models:controllable_dialogue/convai2_finetuned_baseline \
211  -wd extrep_2gram:-3.5,extrep_nonstopword:-1e20,intrep_nonstopword:-1e20,nidf:4
212 
213 Here `nidf:4` means using the NIDF WD feature with weight 4.
214 You can use any real number as a weight (positive or negative).
215 
216 **Talk to the response-relatedness WD model (with WD repetition control):**
217 
218  python projects/controllable_dialogue/interactive.py \
219  -mf models:controllable_dialogue/convai2_finetuned_baseline \
220  -wd extrep_2gram:-3.5,extrep_nonstopword:-1e20,intrep_2gram:-1e20,intrep_nonstopword:-1e20,partnerrep_2gram:-1e20,lastuttsim:5
221 
222 Here `lastuttsim:5` means using the response-relatedness WD feature with weight 5.
223 You can use any real number as a weight (positive or negative).
224 
225 Note that this this feature can take a while to load, especially the first time
226 you run it. This is because we have to load the GloVe vectors from file.
227 
228 **Giving the bot a persona**:
229 If you want the bot to have a persona when you talk to it, select one of the
230 lines in `data/controllable_dialogue/personas_validation.txt` and prepend it to
231 your first utterance. Alternatively you can write the persona yourself - but
232 make sure to use the same format.
233 
234 **Viewing top 10 beam search candidates:**
235 If you want to see the top 10 candidates produced by beam search (rather
236 than just the top 1), add the flag `--verbose True`.

238 ## Train a CT model
239 
240 **To train a CT model from scratch:**
241 
242  python projects/controllable_dialogue/train_controllable_seq2seq.py \
243  -mf /path/to/your/modelfile \
244  --control-vars avg_nidf
245 
246 Here we are training a specificity-controlled CT model.
247 
248 **To change control embedding size:** The CT control embedding size will
249 default to 10, but you could include e.g. `--control-embeddingsize 15` if you
250 wanted to change it.
251 
252 **To change number of buckets:** For `avg_nidf`, the number of buckets will
253 default to 10. If you want to use a different number of buckets, first you
254 need to figure out what the NIDF lower bound should be for each bucket. Suppose
255 you want 8 buckets. First run:
256 
257  python projects/controllable_dialogue/get_bucket_lowerbounds.py \
258  --num_buckets 8 \
259  --control-vars avg_nidf
260 
261 and then copy and paste the provided lower bounds into
262 `projects/controllable_dialogue/controllable_seq2seq/controls.py`, similarly to
263 the existing `AVG_NIDF_10BUCKET_LBS`. Then you can train a model with
264 `--control-num-buckets 8`.
265 
266 **To train a CT model on _multiple_ controls:**
267 
268  python projects/controllable_dialogue/train_controllable_seq2seq.py \
269  -mf /path/to/your/modelfile \
270  --control-vars avg_nidf,question
271 
272 Here we are training a model conditioned on specificity and question-asking.
273 
274 **To take an existing non-CT model and finetune it as a CT model:**
275 First, run this command (in this example, taking the ConvAI2-finetuned
276 baseline and adding specificity control):
277 
278  python projects/controllable_dialogue/train_controllable_seq2seq.py \
279  -mf /path/to/your/modelfile \
280  --init-model models:controllable_dialogue/convai2_finetuned_baseline \
281  --add-control True \
282  --control-vars avg_nidf
283 
284 This command will take the parameters saved in `--init-model`, load them in the
285 new model (which has randomly initialized weights for the new CT parameters),
286 and then save that model to the given modelfile (`-mf`). It should be quick.
287 Once that's done, run this command:
288 
289  python projects/controllable_dialogue/train_controllable_seq2seq.py \
290  -mf /path/to/your/modelfile \
291  --add-control False \
292  --control-vars avg_nidf
293 
294 You should see your new CT model training. Note: this is how the models in the
295 paper were trained.

297 ## Look at generated output and automatic metrics
298 
299 Once you have downloaded the data, you will find a
300 directory `wordstat_files` in `data/controllable_dialogue`.
301 The json files in this directory contain the generated output computed on the
302 ConvAI2 validation set, plus the corresponding automatic metrics.
303 Each json file corresponds to a different model configuration.
304 
305 Run the following:
306 
307  cd projects/controllable_dialogue
308  jupyter notebook
309 
310 and then open up `inspect_wordstats.ipynb`. Where it says `models_dir`, enter
311 the path to your `wordstat_files` directory. You will be able to recreate the
312 table of automatic metrics from the paper (Table 6), and explore the models'
313 generated output.
314 
315 ## Save generated output and automatic metrics to file
316 
317 If you want to generate json files like those in the previous section, run a
318 command like this:
319 
320  python projects/controllable_dialogue/eval_wordstat.py \
321  -mf models:controllable_dialogue/control_questionb11e10 \
322  -wd extrep_2gram:-3.5,extrep_nonstopword:-1e20,intrep_nonstopword:-1e20 \
323  --set-controls question:7
324 
325 This will create a json file containing the output and automatic metrics for
326 the provided model configuration (here, question-controlled CT model with z=7
327 and WD repetition control). The script `eval_wordstat.py` always places the
328 json file in the same place as the model file. The script can take a while to
329 complete - so you can set e.g. `--num-examples 512` to generate output on a
330 smaller number of examples.
331 
332 _Note: Due to changes in ParlAI, there might be some small differences between
333 the json file created via this method, and the json files downloadable in the
334 previous section._

336 ## Human Evaluation code, logs, and analysis
337 
338 Human evaluation logs should be downloaded automatically after following the
339 download instructions above. You'll find them in the `evaluation_logs/` folder.
340 
341 A Jupyter notebook which generates the graphs and tables for the human experiments
342 is available in the
343 [project folder](https://github.com/facebookresearch/ParlAI/tree/master/projects/controllable_dialogue).
344 The notebook should be launched from the ParlAI root directory.
345 
346 The code for running your own mechanical turk evaluations is also available in
347 the corresponding
348 [mturk folder](https://github.com/facebookresearch/ParlAI/tree/master/projects/controllable_dialogue/mturk).
349 You will probably want to make changes to the `model_config.py` and `run.py` to change
350 which models are being evaluated, and then you can launch the experiment with:
351 
352 ```
353 python parlai/mturk/tasks/controllable_dialogue/run.py -r 0.9 --count-complete --hobby --max-resp-time 1200 --max-connections 20 -nc 1200 --sandbox
354 ```
355 Change it to `--live` if you're prepared to spend actual currency. The output must be
356 lightly postprocessed to use it with the analysis tools released. If you intend to do
357 this, please file an issue on the
358 [ParlAI GitHub](https://github.com/facebookresearch/ParlAI/).
