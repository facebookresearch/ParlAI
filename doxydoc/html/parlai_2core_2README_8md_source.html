1 The core library contains the following files:
2 
3 - **agents.py**: this file contains a few basic agents which can be extended by your own model
4  - **_Agent_**: base class for all other agents, implements the act() method which receives an observation table and returns a table in response
5  - **_Teacher_**: child of Agent, also implements the report method for returning metrics. Tasks implement the Teacher class
6  - **_MultiTaskTeacher_**: creates a set of teachers based on a "task string" passed to the Teacher, creating multiple teachers within it and alternating between them
7  - create_task_teacher: instantiate a teacher from a given task string (e.g. 'babi:task:1' or 'squad')
8 - **build_data.py**: basic utilities for setting up data for tasks. you can override if your filesystem needs different functionality.
9 - **dict.py**: contains code for building general NLP-style dictionaries from observations
10  - DictionaryAgent: agent which tracks the index and frequency of words in a dictionary, and can parse a sentence into indices into its dictionary or back
11 - **metrics.py**: computes evaluation metrics, e.g. ranking metrics, etc.
12 - **params.py**: uses argparse to interpret command line arguments for ParlAI
13 - **teachers.py**: contains teachers that deal with dialogue-based tasks, as well as data classes for storing data
14  - **_FixedDialogTeacher_**: base class for a teacher that utilizes fixed data
15  - **_DialogTeacher_**: base class for a teacher doing dialogue with fixed chat logs
16  - **_ParlAIDialogTeacher_**: a teacher that implements a simple standard text format for many tasks (non-visual tasks only)
17 - **thread_utils.py**: utility classes/functions for use in Hogwild multithreading (multiprocessing)
18  - SharedTable: provides a lock-protected, shared-memory, dictionary-like interface for keeping track of metrics
19 - **worlds.py**: contains a set of basic worlds for tasks to take place inside
20  - **_World_**: base class for all other worlds, implements `parley`, `shutdown`, `__enter__`, and `__exit__`
21  - **_DialogPartnerWorld_**: default world for turn-based two-agent communication
22  - **_MultiAgentDialogWorld_**: round-robin turn-based agent communication for two or more agents
23  - **_HogwildWorld_**: default world for setting up a separate world for every thread when using multiple threads (processes)
