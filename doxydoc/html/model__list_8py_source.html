21 There are a number of guidelines you should follow in the zoo:
22 
23 - You should choose the best directory name as possible. An input of
24  ``zoo:PROJECTNAME/MODELNAME/FILENAME`` will attempt to use a build script from
25  parlai/zoo/PROJECTNAME/MODELNAME.py.
26 - You should include all of the following fields:
27  * title: the name of the entry:
28  * id: corresponds to PROJECTNAME
29  * description: describe the entry in reasonable detail. It should be at least
30  a couple sentences.
31  * example: an example command to chat with or evaluate the model
32  * result: the expected output from running the model. You are strongly encouraged
33  to make a nightly test which verifies this result.
34  * external_website: if applicable, an external website related to the zoo to
35  link to.
36  * project: if applicable, a link to the project folder. You must have either
37  external_website or project.
38  * example2 and result2 (optional): additional examples to run.
39 
40 - As much as possible, you should try to include two examples: one to generate
41  some key metrics (e.g. from a paper) and one to actually chat with the model
42  using interactive.py. Both should strongly attempt to minimize mandatory
43  command line flags.
