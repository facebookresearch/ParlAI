1 # Internal subfolder
2 
3 By creating an internal subfolder, you can set up your own custom agents and tasks,
4 create your own model zoo, and manage it all with a separate git repository.
5 We've set it up so you can now work inside the ParlAI folder without the risk of accidentally pushing your work to the public ParlAI repo.
6 We've also added some convenient shortcuts to mirror what we provide in the `parlai` folder.
7 
8 ## How to
9 
10 Start by creating a new folder named parlai_internal and copying the contents of this folder.
11  (We've added this to .gitignore already.)
12 
13 ```bash
14 cd ~/ParlAI
15 mkdir parlai_internal
16 cp -r example_parlai_internal/ parlai_internal
17 cd parlai_internal
18 ```
19 
20 We've ignored this folder, but that's it. If you want to set this up as
21 a separate git repository (e.g. for version control) you can follow the standard steps
22 for creating a new repo (feel free to do this however you prefer).
23 
24 ```bash
25 git init
26 git add .
27 git commit -m "Initialize parlai_internal"
28 ```
29 
30 You can connect this to a new github repository if desired.
31 [Create a new repo](https://github.com/new) (you don't need to initialize
32 with a README), and then follow the instructions to push
33 an existing repository from command line.

36 ## Some features
37 
38 We also provide a number of shortcuts which mirror the public repo.
39 
40 You can do `from parlai_internal.X.Y import Z` to use your custom modules.
41 
42 Additionally, you can invoke your internal model agents from command line with `-m internal:model`.
43 Providing this argument will cause the parser to look for `parlai_internal.agents.model.model.ModelAgent`.
44 As an example, we provide `parlai_internal/agents/parrot/parrot.py`. You could call
45 (from the top-level ParlAI folder):
46 
47 ```bash
48 python examples/display_model.py -t babi:task10k:1 -m internal:parrot
49 ```
50 
51 Similarly, you can add private tasks under a tasks folder here and invoke them with `-t internal:taskname`.
52 The parser will look for `parlai_internal.tasks.taskname.taskname.DefaultTeacher`.
53 
54 You can even create your own model zoo of pretrained models. `parlai_internal/zoo/.internal_zoo_path`
55 needs to be modified to contain the path to the folder containing all of your models. Once
56 you've done that, you can use those models by simply adding `-mf /rest/of/modelfilepath`.
57 For example, if you change `.internal_zoo_path` to be `/private/home/user/checkpoints`
58 and you have a model at `/private/home/user/checkpoints/model_file/model`, you could use `-mf izoo:model_file/model`.
59 
60 And you can use as many of these in combination as you would like. For instance, to evaluate a model file that uses
61 an internal agent definition on an internal task, you would do:
62 
63 ```bash
64 python examples/eval_model.py -t internal:taskname -m internal:model -mf izoo:model_file/model
65 ```
